
### Pre-Delivery Checklist (Embedded in All Skills)

Every skill runs this validation before returning output:

**Visual Quality:**
- [ ] No emoji icons (SVG only via Lucide/Heroicons)
- [ ] Icons from consistent set
- [ ] Brand logos verified (Simple Icons)
- [ ] No layout shift on hover
- [ ] Icon sizing consistent (w-6 h-6)

**Interaction:**
- [ ] `cursor-pointer` on ALL clickables
- [ ] Hover feedback clear (color/opacity, NOT scale)
- [ ] Transitions 150-300ms
- [ ] Focus states visible (ring-2 ring-primary)
- [ ] No hover-only interactions on mobile

**Light/Dark Mode:**
- [ ] Light mode readable (#0F172A text, contrast â‰¥4.5:1)
- [ ] Light glass visible (bg-white/80+)
- [ ] Dark mode readable (#F8FAFC text)
- [ ] Dark glass visible (bg-black/40+)
- [ ] Borders visible in both modes
- [ ] BOTH modes tested before delivery

**Layout:**
- [ ] Floating navbar (top-4 NOT top-0)
- [ ] Content padding accounts for fixed elements
- [ ] Consistent max-width (max-w-6xl or max-w-7xl)
- [ ] Responsive breakpoints tested (375px, 768px, 1024px, 1440px)
- [ ] No horizontal scroll on mobile
- [ ] Z-index scale used (10, 20, 30, 50)

**Accessibility (WCAG 2.1 AA):**
- [ ] All images have alt text
- [ ] Icon-only buttons have aria-label
- [ ] Form inputs have labels (htmlFor + id)
- [ ] Color not the only indicator
- [ ] Keyboard navigation works
- [ ] Focus states visible
- [ ] Touch targets â‰¥44x44px
- [ ] prefers-reduced-motion respected

**Performance:**
- [ ] Images: WebP, srcset, lazy loading
- [ ] Animations: transform/opacity only (NO width/height/top/left)
- [ ] CLS < 0.1
- [ ] 60fps animations (verified in Chrome DevTools)
- [ ] Lighthouse Mobile >90, Desktop >95
- [ ] No unused dependencies

### Awwwards-Level Validation

**The 7-Word Magic Formula Check:**
> "Cinematic scroll parallax bold oversized interactive smooth"

Every web artifact must pass:
- [ ] Scroll-triggered animations (GSAP or Framer Motion)
- [ ] Parallax effects (Lenis or Locomotive Scroll)
- [ ] Bold typography (â‰¥48px headlines, variable fonts)
- [ ] Oversized visual elements (hero images, 3D objects)
- [ ] Interactive micro-animations (hover, click, scroll)
- [ ] Smooth 60fps performance (no jank)

**Anti-Pattern Detection:**
- [ ] No emoji icons anywhere
- [ ] No layout shift on hover (CLS validation)
- [ ] No poor contrast (automated check)

- [ ] No generic templates or carousels
- [ ] No auto-playing audio

## ðŸš€ Getting Started

### 1. Initial Setup

```bash
# Clone Synthia repository
git clone https://github.com/yourusername/synthia.git
cd synthia

# Copy environment template
cp master.env .env

# Update .env with your actual API keys and tokens
nano .env

# Start Docker services
docker-compose up -d

# Initialize multi-agent coordination (if using)
export AGENT_NAME="SynthiaOrchestrator"

# Initialize Beads task management
bd init

# Verify setup
docker ps  # Should show all Synthia services running
bd ready   # Should show task management initialized
resource://inbox/$AGENT_NAME?project=$(pwd)  # Should return empty inbox
```

```bash
# All skills are in /skills directory
ls skills/

# Each skill is self-contained markdown with:
# - skill_id
# - purpose and when_to_use
# - inputs/outputs specification
# - tools_&_integrations
# - project_specific_guidelines
# - example_interactions
```
